# 关卡地图系统 — 实现总结

> 本文档汇总「小猪推推乐」主界面关卡地图系统的当前实现状态，涵盖按钮三态、路径生成、点击进关与按压反馈。
> 最后更新：2026-07-18

---

## 一、系统概述

主界面是一条**程序化生成的蜿蜒路径**，上面按进度分布关卡按钮，可上下滚动浏览全部关卡。
- 点击**已解锁**关卡直接进关；**当前关**走开始流程（消耗体力）；**未解锁**关卡锁定不可点。
- 路径锁定在屏幕中央 50% 带，**左右各空 1/4 屏宽**留给风景。
- 满尺寸按钮（`_btnScale = 1`），每摆臂 2 个（左-右极值交替）的蛇形布局。

## 二、关键文件与职责

| 文件 | 职责 |
|------|------|
| `js/ui/LevelMap.js` | 路径生成、滚动、按钮布局、点击命中、按压反馈、绘制调度 |
| `js/ui/widgets/LevelButton.js` | 单关按钮三态绘制（星级花、数字、呼吸环） |
| `js/define/LevelMapConfig.js` | 布局参数（关卡数、间距、路径样式、滚动手感） |
| `js/core/GameEngine.js` | 菜单触控路由、进关流程、给地图注入 `onSelectLevel` 回调 |
| `js/anim/ButtonPress.js` | 项目通用按压反馈（按下缩 0.95、松手回弹），关卡钮已接入 |

> 纯 JS 改动，微信开发者工具**热重载即可生效，无需重编译打包**（仅新增/替换 png 时才需重编译）。

## 三、按钮三态（LevelButton）

| 状态 | 视觉 | 可点？ |
|------|------|--------|
| `cleared`（已通关） | 满尺寸 117×115 最终版按钮，按真实星级画花 + 中心数字 | ✅ |
| `current`（当前关） | 灰圆图 + 居中数字 + **绿色 `#3FB6A8` 呼吸环**（唯一常驻呼吸表现） | ✅（耗体力） |
| `locked`（未解锁） | 灰圆图 + 居中数字，无呼吸环 | ❌ |

### 星级花规则
- **1~3 星** → N 朵普通黄花（`drawFlower` `colored:false`）。
- **4 星**（魔法棒成就，`stars>=4`）→ 3 朵彩花（`colored:true`）。
- 花位置固定：中 `(58.5, 89.5)` / 左 `(26.69, 91.69)` / 右 `(90.31, 91.69)`，带轻微阴影。

### 关卡数字（locked / current）
- 居中于灰圆中心 `(58.5, 61)`。
- 颜色 `#7E8B97`、字号 `32px`、`font-weight 400`、`大宝桃桃体`（由 `wx.loadFont(dudu.ttf)` 注册为 `Theme.font.family`）。

## 四、路径生成（LevelMap）

### 程序化蛇形
- 水平：`wx = xMid + amp * Math.sin(t * π * (N-1) + π/2)`，`t = i/(N-1)`。
  - 相邻关落在左右极值交替 → 每摆臂 2 个（左-右），水平错开 ≈ 2·amp，永不水平重叠。
- 竖直：`wy = topReserve + (N-1-i) * gap`，`gap = levelGap * s`。
- 路径：通过 `_buildRoadPath(pts)` 用 **Catmull-Rom 样条**转三次贝塞尔，精确穿过每个按钮中心点。

### 中央带 + 风景约束
- `margin = SCREEN_WIDTH * 0.25`；`xMin = margin + halfRoad`，`xMax = W - margin - halfRoad`。
- `xMid = (xMin+xMax)/2`，`spacing = (xMax-xMin)/2`，`amp = spacing`（满幅）。
- 路边缘最远 `xMid + amp + halfRoad` ≤ `0.75*W`，**不侵入风景带**。

### 底部 30px 留白（短屏可见第一关）
```
bottomReserve = (65+86)*s + 57.5*_btnScale + 30
```
- 开始按钮 180×86，底边距屏底 `65*s` → 顶部距屏底 `(65+86)*s`。
- 公式保证「第一关按钮底 ↔ 开始钮顶」恒为 **30px**，与屏幕高无关。

## 五、点击与进关

### 进度判定（`_onTapLevel`）
```
state = (idx <= lastIdx) ? 'cleared'
      : (idx === frontier) ? 'current' : 'locked'
```
- `locked` → 忽略（不进按压态）。
- `current` → 走原开始流程（消耗体力，同开始按钮）。
- `cleared` → 直接进关，**不耗体力**。

### 进关流程（GameEngine）
- 地图注入 `onSelectLevel(index, state)` 回调：`cleared` → `startLevelByIndex(idx, false)`；`current` → `_onClickPlayBtn`。
- `startLevelByIndex(idx, consumeStamina)`：写 `currentLevel/currentLevelIndex` → 播菜单出场动画（并行加载关卡）→ 进 `playing`，与开始按钮流程一致。

## 六、按压反馈（接入 ButtonPress）

**根因**：关卡钮此前未接项目通用按压机制，且为避开选中环曾被做成「一碰即进」，跳过了按压过程，看着像没反应。

**修法**：关卡钮接入 `js/anim/ButtonPress.js`，与开始/设置按钮手感一致。
- 按下**可点关**（cleared/current）→ 整钮缩到 0.95、本回合**不滚地图**（按钮按着时该缩着）。
- 松手未拖动 → 才进关。
- 拖动 >12px → 取消按压、转为滚地图。
- `locked` → 不进按压态（禁用按钮本就不响应）。
- `_drawLevelButton` 围绕按钮中心把整钮（图+花+数字+呼吸环）一起缩放，松手回弹。

## 七、防回归纪律（已写入 MEMORY.md）

1. **菜单触控路由**：所有模态面板（商店 / 体力广告 / 设置）的优先检查**必须位于关卡地图手势闸门之前**。否则面板居中弹出时按钮会集体失灵（地图手势误吃触摸）。
2. **派生量/镜像字段**：断点续玩等状态改动先问「是镜像字段还是派生量」，再决定存盘与续玩恢复方式。

## 八、可调参数（旋钮）

| 想要的效果 | 改哪里 |
|-----------|--------|
| 波形更平 / 更甩 | `LevelMap.js` 约 121 行 `amp = spacing`（改 `spacing * 0.9` 更平；满幅 `1.0` 更甩） |
| 上下更紧 / 更疏 | `LevelMapConfig.levelGap`（当前 130；想更紧调到 >115 即可防上下重叠） |
| 开始钮 / 风景留白 | 开始钮 180×86、底边距屏底 `65*s`；改需同步 `bottomReserve` 公式 |

**约束口诀**：`amp` 与按钮宽不能让「水平错开 < 钮宽」或「垂直间距 < 钮高」，否则钮会叠。

## 九、验证记录

- 所有改动均过 `node --check` 语法校验。
- **路径布局几何模拟**（375 / 414 / 320 多种屏宽 + 600 / 568 短屏）：点亮模式 `RLRLRLRLRLR`、**零重叠**、路不侵风景带、30px 留白恒成立。
- **点击状态机桩测试**：
  - 点 cleared → 选中按压态 + 触发缩放动画 → 松手回调 `onSelectLevel(0,'cleared')`。
  - 点 locked → 不进按压态（忽略）。
  - 小幅移动（<12px）→ 不滚地图、保持按压。
  - 大移（>12px）→ 取消按压、转滚动。
  - 切 `gameState='playing'` → 选中/进关态自动清理，呼吸环随菜单淡出不残留。
