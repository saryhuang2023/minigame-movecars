# 场外求助（Help-a-Friend）系统策划案

> 版本：v2.0（2026-07-21 落定，全部功能已实现并通过测试）
> 范围：发起求助 → 好友协助 → 回放复盘 完整闭环

---

## 一、功能定位

**一句话**：玩家关卡内卡关 → 把残局快照发给微信好友 → 好友在棋盘上操作帮忙推猪 → 玩家看回放复盘解法。

本系统是"小猪推推乐"的 **社交裂变 + 互助破局** 双驱动功能，不改变核心消除玩法，而是用微信社交关系链把"卡关 → 流失"转化为"分享 → 拉新/回流 → 破局"。

---

## 二、三端流程总览

```
┌──────────────┐   wx.shareAppMessage    ┌──────────────┐   wx.shareAppMessage    ┌──────────────┐
│  玩家（发起）  │ ──── helpKey ────────▶│  好友（协助）  │ ──── 录制回传 ────────▶│  玩家（回放）  │
│  playMode     │                        │  playMode     │                        │  playMode     │
│  = normal     │                        │  = assist     │                        │  = replay     │
└──────┬───────┘                        └──────┬───────┘                        └──────┬───────┘
       │                                       │                                       │
  ① 点"求助"按钮                         ② 点分享卡片                         ⑤ 点"奖杯"→回放
  ② 拍当前残局快照                       ③ 在残局棋盘上操作                   ⑥ 自动重演好友解法
  ③ 发微信分享卡片                       ④ 通关/提前结束→回传录制             ⑦ 查看跑出几头猪
  ④ 等好友回传                           ⑤ 可再来一次（最多3人）               ⑧ 从此解法中学习破局
```

**三个子模式**（共享同一 `playing` 游戏态，靠 `playMode` 区分 UI/行为）：

| 模式 | playMode | 谁来操作 | 是否录制 | 是否落库 | 关卡数据来源 |
|------|----------|---------|---------|---------|------------|
| 正常游玩 | `normal` | 玩家自己 | 可选（提示录制） | 是 | 工程关卡文件 |
| 协助好友 | `assist` | 协助者 | 是（必录） | 否（仅回传云端） | 残局快照还原 |
| 回放复盘 | `replay` | 无（自动播放） | 否 | 否 | 残局快照还原 |

## 三、功能详情

### 3.1 发起求助（Flow A）

**入口**：关卡内底部工具栏「求助」按钮（每关 2 次）。

**流程**：
1. 点击求助 → 取玩家昵称头像（`wx.createUserInfoButton` 手势授权，符合微信隐私合规）
2. 冻结当前棋盘状态为快照（棋盘参数 + 所有猪的位置/朝向/长度 + 已用步数 + 剩余额外步数）
3. 压缩快照 → 云端创建 `help_requests` 记录 → 获得唯一 `helpKey`
4. 弹出微信分享卡片（标题「帮我过这关！」，携带 `helpKey`）
5. 求助次数 -1

**设计要点**：
- 同一关可多次求助（每次新建一条，最多每人 3 个协助者接同一 key）
- 快照是**时间点冻结**——玩家发完后自己继续玩不影响已发出快照
- 分享卡片可继续转发给更多好友（同一 helpKey，assists 上限 3 人）
- 求助中屏蔽棋盘操作（`_externalHelpBusy` 标志位），防止流程中途误触

### 3.2 好友协助（Flow B）

**入口**：微信聊天中点分享卡片 → 冷启/热启小游戏 → 主菜单弹协助邀请卡 → 进协助关。

**流程**：
1. 解析卡片中的 `helpKey` → 云端拉取残局快照 → 还原棋盘
2. 显示左上角请求者信息：「来自好友的求助」+ 圆形头像 + 昵称
3. 好友在残局棋盘上操作（与正常游玩相同的手感，推猪/碰撞规则不变）
4. 操作全程自动录制（以棋盘数据坐标为基准，跨设备兼容）
5. 三种结束方式：

| 结束方式 | 触发 | 结束面板文案 | 按钮 |
|---------|------|------------|------|
| 通关 | 所有猪逃脱 | 「恭喜你通关了，快发给好友吧」 | 下次再说 / 发给好友 / 再来一次 |
| 提前结束 | 点「就帮到这吧」 | 「差不多了，帮一点也是帮」 | 同上 |
| 步数用完 | 剩余步数归零 | 「哎呀，步数不够了，就只能帮到这了」 | 同上 |

6. 点「发给好友」→ 录制上传云端 → 弹微信分享卡片 → 中钮变为「去玩玩」（文案「该帮的也帮了，该我玩了」）→ 点之回主菜单

**UI 表现纪律**（与正常关卡的区别）：
- 隐藏：设置按钮、关卡标题、金币控件、树枝/小虫/花朵进度条、装饰图（`bg_deco_718`/`level_brush`）、木质底板
- 保留：剩余步数（RightStepWidget）、猪头计数器（PigCounter）
- 左上角：请求者头像 + 昵称 + 「来自好友的求助」（白色描边文字，顶到状态栏下边缘）
- 左下角：录制指示器 `rec...`（红脉冲圆点）+ 提示指示器 `hint...`（琥珀灯泡，协助关不记提示）
- 底部面板：弹窗式三宫格（与设置面板同款背景），标题「协助完成」，底部三钮：左 `btn_home`（下次再说）、中蓝色「发给好友」、右 `btn_again`（再来一次）
- 按钮按压反馈：全部标准按压效果（按下缩小 0.94 + `button_click` 音效）

**数据纪律**：
- 协助关**不落库**（不写 bestScore/star/金币/云端同步）
- 协助关**不记提示**（`_hintMerged = true` 阻断）
- 录制坐标以棋盘数据坐标为基准（`h.x - boardOffsetX, h.y - topBarH - boardOffsetY`），**跨设备回放不乱位**

### 3.3 进入分支处理

好友点卡片进协助关时，云端返回当前求助状态，客户端据此弹对应分支：

| 状态 | 弹窗类型 | 行为 |
|------|---------|------|
| 正常可协助 | 无弹窗 | 直接进残局开始操作 |
| 已协助过（本人） | `wx.showModal` 确认/取消 | 「这局您已经协助过了，重新协助将覆盖掉旧的协助记录」；确认→进关重录，取消→回菜单 |
| 协助者已满（非本人） | 单钮「好的」 | 「协助者已超上限 3 人」→ 回菜单 |
| 请求已过期/不存在 | 单钮「好的」 | 「请求已过期」/「该协助已删除」→ 回菜单（toast） |
| 数据异常 | 单钮「好的」 | 「该协助异常（{code}）」→ 回菜单 |

**技术要点**：`alreadyAssisted` / `isFull` 标志位由云函数 `getHelpRequest` 计算返回（code 体系 0/1/2/4/-1 不变），客户端判定分支。回放读取路径不受影响（不判 `alreadyAssisted`/`isFull`）。

### 3.4 回放复盘（Flow C）

**入口**：主菜单「奖杯」按钮 → 求助记录面板 → 点某条记录的「回放」按钮。

**流程**：
1. 进入回放关 → 正中显示 5 秒倒计时圆圈（1~5）
2. 倒计时结束 → 自动播放好友的录制操作（小猪按录制时序移动）
3. 回放过程中：「返回」按钮底部居中，下方显示「回放中…」
4. 回放结束后：「回放已结束」文字不消失，按钮变为「返回」+「再看一次」水平并列居中
5. 点「再看一次」→ 走完整过场（圆形虹膜过渡 → 销毁当前关卡 → 重新进入 → 重新倒计时 → 重新回放）

**UI 纪律**：
- 回放关与协助关共用 UI 隐藏规则（无树枝/装饰/底板）
- 左上角显示「来自好友的帮助」+ 头像 + 昵称
- **全程屏蔽玩家棋盘输入**（不能拖动小猪），仅回放自身合成触控放行
- 左下角不显示 rec/hint 指示器

### 3.5 求助记录面板

**入口**：主菜单左侧「奖杯」按钮。

**结构**：双页签切换式面板（与设置面板同款弹出风格）。

| 页签 | 内容 | 排序 | 卡片要素 |
|------|------|------|---------|
| 我发出的 | 我发起的求助及协助记录 | 按发布时间倒序 | 日期·第 X 关 / 协助者头像+昵称 / 力度（通关/跑出 X 头猪） / 亲密度 / 回放钮 |
| 我协助的 | 我帮他人的协助记录 | 按我的协助时间倒序 | 日期·第 X 关 / 请求者头像+昵称 / 力度 / 亲密度 / 回放钮 |

**特殊规则**：
- 无协助者时：显示占位行「暂无好友协助」，按钮变为蓝色「重发」（重新 `wx.shareAppMessage` 发卡片）
- 有协助者时：按钮为绿色「回放」
- 顶部常驻说明：`*仅保留 7 天内的协助信息`
- 亲密度口径：双向聚合（我发 TA 助 + TA 发我助），在客户端按好友 openId 聚合计数

## 四、核心参数与约束

| 参数 | 值 | 说明 |
|------|-----|------|
| 每关求助次数 | 2 次 | 与 +3 步数、提示 共用「每关」配额 |
| 每求助可协助人数 | 最多 3 人 | 云端 `assists[]` 数组上限 |
| 协助记录保留期 | 7 天 | 超期云端自动清理，面板自然不展示 |
| 同一人重复协助 | 可覆盖 | 旧录制被新录制覆盖，不占额外名额 |
| 快照自包含性 | 完整棋盘 + 猪数据 | 不依赖关卡云端版本，离线可复现 |
| 录制坐标基准 | 棋盘数据坐标 | 跨设备回放不乱位 |
| 协助关体力消耗 | 不消耗 | 协助不消耗体力 |
| 协助关金币 | 不产出 | 不记金币、不落库 |

## 六、数据模型（云端）

### 集合 `help_requests`

| 字段 | 类型 | 说明 |
|------|------|------|
| `helpKey` | string | `hk_` 前缀，全局唯一 |
| `requesterOpenId` | string | 发起人 openid |
| `requester` | object | `{ nickName, avatarUrl }` |
| `levelName` | string | 关卡文件名 |
| `snapshot` | string | BASE64(DEFLATE(棋盘快照)) |
| `status` | string | `open` / `full` / `expired` |
| `assists` | array | 最多 3 条，每项 `{ assistantOpenId, assistant{nickName,avatarUrl}, recording, result{escapedPigs,totalPigs}, recordedAt }` |
| `createdAt` | date | 服务端时间 |
| `expiresAt` | date | `createdAt + 7 天` |

## 七、隐私与合规

- 昵称/头像：必须由玩家主动手势授权（`wx.createUserInfoButton`），发起和协助各授权一次
- 快照数据不包含任何用户个人信息，仅含游戏棋盘状态
- 云端 `help_requests` 7 天后自动过期清理
- 所有跨用户读取均走云函数（服务端 admin 权限），客户端不直接跨 openid 查询

---

## 八、与核心玩法的关系

- **不影响正常关卡**：`playMode === 'normal'` 时行为完全不变
- **不消耗核心资源**：协助不扣体力、不产金币、不推进关卡进度
- **不污染排行榜/成就**：协助关不写 bestScore/star/云端同步
- **社交裂变独立**：求助/协助/回放为独立闭环，可随时增删而不影响核心消除玩法
