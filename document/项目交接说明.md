## 3. 数据结构

## 4. 关键系统

### 金币（客户端权威）
- `GoldSystem` 单例；`calculateReward = pigCount`；客户端权威（`savePlayerData` 上报），不做服务端防刷。`cloud.settleLevel` 是死代码，保留勿接。
- 🚫 步数奖励防回归：金币落地 `setData` 须 `if(!this._goldSettled)` 守卫；`_startStepBonusTicker` 后等~1400ms 再弹 VictoryPopup。

### 星级（2026-07-16 新方案落地）
- 详见 `document/关卡星级算法-设计方案.md`
- **每步=1分**（`_scoreBonusRemaining = stepBonus`，已从旧的×2 改为×1）
- 难度3档：`difficulty` 字段优先，否则按关卡ID（≤20简单/≤100标准/>100难）
- 默认阈值：`minSteps = N×{1.3/1.4/1.5}`，可省=`totalSteps−minSteps`，`s3 = N + 可省×0.55`，`_starScores = [s3/4, s3×2/3, s3, N+可省×0.80]`（1星=3星×1/4，2星=3星×2/3，均取整）
- `getStarTier(achievedScore, _starScores)` 判定逻辑不变；`achievedScore = escapedCount + scoreBonusRemaining`
- 星级是玩家进度，走 `players` 集合（**绝不**走关卡 JSON bestScore）；本地 key `levelStars`
- 编辑器星级浮层：新增难度3按钮（简单/标准/难），切换即重算默认；"恢复默认"按当前难度算

### 提示系统
- 编辑器增删精灵清空所有提示（调整/旋转不清空）；试玩推猪实时写 hint；通关 `_mergeAndUploadHints()` 全量上传
- 无 hint 隐藏提示按钮

### 步数/失败（RightStepWidget）
- `remaining = threshold - steps`；失败 `_checkFail()` 在 onTouchEnd 计步+tryPushPig 后；通关优先失败
- 数字滚动：增减都滚，常规里程表方向（变大旧上出新下进、变小旧下出新上进），1000ms/22px

### 云端
- 在用集合：`players`/`levels`/`bug_reports`
- `savePlayerData`/`getPlayerData`；通关推 `lastLevelIndex`；星级按 key 取 max 合并
- 云函数超时3秒，须 config.json timeout:60

## 7. 🚫 工程禁区（务必遵守）

- **UIComponent 是 ES6 class**：widget 必须 `class X extends UIComponent`，禁 `UIComponent.call`
- **require 路径坑**：`AssetPreloader.js` 在 `js/ui/`。从 `js/effects/`、`js/game/` 引用必须 `require('../ui/AssetPreloader.js')`（往上两级）；`js/ui/widgets/` 内用 `require('../AssetPreloader.js')`。新建 effects 文件对照同目录 `CoinFlyEffect.js`。
- **本地图片**须 AssetPreloader.register→preload→get（禁 wx.createImage）
- **Figma 坐标**：带父 frame 整体组件按父 frame 原点换算；散件屏幕坐标。用户说换算就换算、说屏幕就屏幕
- **已验收 UI 不擅自改**：一次只动当轮目标，改相邻组件前先问
- **绘制坐标与触摸命中区必须成对同步**（曾因 y:26 vs y:78 致按钮点不到）
- **VictoryPopup 遮罩是半透明**（rgba(0,0,0,maskAlpha)，maskAlpha 为小数），下层 UI 透过遮罩仍可见，勿凭 fillRect 判不可见
- **圆角矩形 radius** 必须 `Math.min(r, floor(w/2), floor(h/2))`
- 禁裸引用 `global`（用 GameGlobal 探测）
- `PlayingDefine.PLAY.ENTRANCE` 键名是 `TOTAL`（非 TOTAL_DURATION）
- 步数判定用 `findHeadHole()` 直比头孔索引
