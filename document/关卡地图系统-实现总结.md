# 关卡地图系统 — 实现总结

> 本文档汇总「小猪推推乐」主界面关卡地图系统的当前实现状态，涵盖按钮三态、路径生成与无缝渲染、点击进关与按压反馈。
> 最后更新：2026-07-19

---

## 一、系统概述

主界面是一条**跨段拼装的蜿蜒路径**，上面按进度分布关卡按钮，可上下滚动浏览全部关卡。
- 点击**已解锁**关卡直接进关；**当前关**走开始流程（消耗体力）；**未解锁**关卡锁定不可点。
- 路径由三段式路径图（`road_0` / `road_1` / `road_2`）循环拼装，自底向上延伸；L1 落地图最底，编号自下向上递增。
- 路径**全程用 Canvas 程序化连续曲线绘制**（`_renderRoad`），不依赖任何 road PNG 烘焙路径 —— 这是「跨段 100% 无缝」的根本保证。
- 顶端余量段吸收多出的关卡数，保证底部 `road_0` 始终满 11 钮。

## 三、按钮三态（LevelButton）

| 状态 | 视觉 | 可点？ |
|------|------|--------|
| `cleared`（已通关） | 满尺寸 117×115 最终版按钮，按真实星级画花 + 中心数字 | ✅ |
| `current`（当前关） | 灰圆图 + 居中数字 + **绿色 `#3FB6A8` 呼吸环**（唯一常驻呼吸表现） | ✅（耗体力） |
| `locked`（未解锁） | 灰圆图 + 居中数字，无呼吸环 | ❌ |

### 星级花规则
- **1~3 星** → N 朵普通黄花（`drawFlower` `colored:false`）。
- **4 星**（魔法棒成就，`stars>=4`）→ 3 朵彩花（`colored:true`）。
- 花位置固定：中 `(58.5, 89.5)` / 左 `(26.69, 91.69)` / 右 `(90.31, 91.69)`，带轻微阴影。

### 关卡数字（locked / current）
- 居中于灰圆中心 `(58.5, 61)`。
- 颜色 `#7E8B97`、字号 `32px`、`font-weight 400`、`大宝桃桃体`（由 `wx.loadFont(dudu.ttf)` 注册为 `Theme.font.family`）。

## 四、路径生成与无缝渲染（LevelMap）

### 4.1 段拼装架构 `_buildLevels`
- 设计画布 `designWidth = 393`；统一缩放 `k = roadTargetW / 845 * (SCREEN_WIDTH / 393)`，`roadTargetW = 270`（`road_0` 自然宽 845 在 393 画布上目标显示宽 270）。
- 钮坐标 `LevelMapConfig.roadButtons.{road_0, road_1, road_2}`（图像像素空间，已 `resample_uniform.js` 等弧长均匀化）。
- **段序列（底→顶）**：`road_0(11)` → 其上 `road_1` / `road_2` 交替循环；顶端段吸收余量（`count < 全长`）。
  - **余量段取路径「末端」count 个钮**（`slice(fullLen - count)`），使衔接钮为路径真终点。若取「首」count 钮，衔接点落在路径中段 → 造成 L39↔L40 式异常大缝。
- **跨段补白（局部均值）**：每段放置后，按两邻段近缝处段内间距均值补白
  `targetImg = (prevB.lastGap + curB.firstGap) / 2`（含两段 `roadLeft` 偏移差的 2D 距离反解竖直补白），使跨段相邻钮间距与段内一致（不再出现 L11↔L12 比段内更挤）。
- 构建完 `reverse()`：L1 落最底、编号自下向上递增；`index === 数组下标` 约定不变。
- **末尾留白 `trailBottom`（默认 100 design px）**：整体平移所有钮与段 `shiftWorld = trailBottom * s - minLevelY`。**不影响** `road_0 ↔ 开始钮` 间隙（该间隙为 `segY` 之上的附加项）。

### 4.2 间距三修（用户反馈闭环）

| 衰因 / 修复 | 说明 |
|-----------|------|
| **L26↔L27 过近** | `road_2` 自然坐标不均匀 → 用 `resample_uniform.js` 等弧长均匀化钮坐标，段内间距一致。 |
| **L39↔L40 过远（1229px 异常）** | 余量段原取「首」count 钮，衔接点在中段 → 改取「末端」count 钮，衔接为路径真终点。 |
| **末尾无空间** | 顶关原 flush 在 content 顶 → 加入 `trailBottom` 整体平移，顶关上方恒留 100px。 |

### 4.3 无缝路径渲染 `_renderRoad`（程序化连续曲线）

**根因**：原 `road_0/1/2` PNG（透明底 + 绿虚线）跨段无法无缝 —— 不同图路径端点 x 不对齐，且段间 `pad` 空隙无图像绘制 → 路径物理断线（11↔12、39↔40 等多处断开）。

**方案**：放弃 PNG 烘焙路径，改用 **Canvas 程序化连续曲线**：

1. 取所有 `_levels[].(x, worldY)` 为控制点；
2. **Chaikin 角点切割** `cfg.roadSmoothIters`（默认 2）：把每个尖角替换为一段圆弧，**消除锐角**（高速翻车式），保留首尾端点（首关 / 末关精确落在路端），整体路由不变（仍依次经过各关）；
3. **Catmull-Rom** 插值（步长 `18px`）生成密集路径点，**单次 `stroke()`** 一条连续曲线 → **零断点**；
4. 两层描边（同一组路径点）：
   - ① **白色半透明路身**：`rgba(255,255,255,0.30)`，宽 `50` design px，实线、`round` cap/join；
   - ② **绿色虚线中线**：`#7ECB50`，宽 `5.5` design px，虚线 `[cfg.roadDash[0], cfg.roadDash[1]]`（默认 `[6, 16]` = 短线段 + 稀疏间隔）。

> 为什么能无缝：路径是**一条**从首关连到末关的数学曲线，段与段之间没有任何图像拼接边界，自然不存在断点。

### 4.4 验证（模拟，N=47）

| 指标 | 值 |
|------|----|
| 段内 / 跨段间距 | L11-12≈105 / L26-27≈104 / L39-40≈104 / L46-47≈104 px（统一 ~104） |
| 顶关上方留白 | 精确 **100 px** |
| 路径点数 | 275+，零 NaN |
| 每关是否落在路径上 | 最大偏移 **0.00 px**（精确穿过；Chaikin 后偏离 ≤17px 仍压路面） |
| 最尖单弯转角 | 29.5°（iters=0）→ **18.8°**（iters=2）；iters=3 → 13.9° |
| 关卡中心 ↔ 路径最大偏移 | 17.2px（iters=2）< 路半宽 25px → 钮仍压路面 |

## 五、点击与进关

### 进度判定（`_onTapLevel`）
```
state = (idx <= lastIdx) ? 'cleared'
      : (idx === frontier) ? 'current' : 'locked'
```
- `locked` → 忽略（不进按压态）。
- `current` → 走原开始流程（消耗体力，同开始按钮）。
- `cleared` → 直接进关，**不耗体力**。

### 进关流程（GameEngine）
- 地图注入 `onSelectLevel(index, state)` 回调：`cleared` → `startLevelByIndex(idx, false)`；`current` → `_onClickPlayBtn`。
- `startLevelByIndex(idx, consumeStamina)`：写 `currentLevel/currentLevelIndex` → 播菜单出场动画（并行加载关卡）→ 进 `playing`，与开始按钮流程一致。

## 六、按压反馈（接入 ButtonPress）

**根因**：关卡钮此前未接项目通用按压机制，且为避开选中环曾被做成「一碰即进」，跳过了按压过程，看着像没反应。

**修法**：关卡钮接入 `js/anim/ButtonPress.js`，与开始/设置按钮手感一致。
- 按下**可点关**（cleared/current）→ 整钮缩到 0.95、本回合**不滚地图**（按钮按着时该缩着）。
- 松手未拖动 → 才进关。
- 拖动 >12px → 取消按压、转为滚地图。
- `locked` → 不进按压态（禁用按钮本就不响应）。
- `_drawLevelButton` 围绕按钮中心把整钮（图+花+数字+呼吸环）一起缩放，松手回弹。

## 七、防回归纪律

1. **菜单触控路由**：所有模态面板（商店 / 体力广告 / 设置）的优先检查**必须位于关卡地图手势闸门之前**。否则面板居中弹出时按钮会集体失灵（地图手势误吃触摸）。
2. **派生量/镜像字段**：断点续玩等状态改动先问「是镜像字段还是派生量」，再决定存盘与续玩恢复方式。
3. **路径必须程序化连续**：关卡路径渲染**只用** `_renderRoad` 的程序化曲线，**禁止回退到 road PNG 烘焙路径**——PNG 跨段物理上无法无缝，会导致 11↔12、39↔40 等断点复发。
4. **余量段取末端钮**：`_buildLevels` 中部分段必须用 `slice(fullLen - count)`（末 count 钮），不得改回「首 count 钮」，否则跨段缝距异常。
5. **重编译铁律**：改 JS / 配置后务必清缓存 + 重编译打包，否则看的是旧资源。

## 八、可调参数（旋钮）

| 想要的效果 | 改哪里 |
|-----------|--------|
| 段内钮间距（更均匀） | `LevelMapConfig.roadButtons` → 跑 `tools/resample_uniform.js` 重新均匀化 |
| 跨段缝距 | `_buildLevels` 局部均值 padding（一般无需手改；异常时查 `lastGap`/`firstGap`） |
| 末尾留白 | `LevelMapConfig.trailBottom`（默认 100） |
| 路径圆角程度 | `LevelMapConfig.roadSmoothIters`（2 够；3 更圆但偏离中心略多） |
| 绿虚线线段长 / 间隔稀疏 | `LevelMapConfig.roadDash`（默认 `[6, 16]`：线段 6 / 间隔 16，design px） |
| 白路身宽度 / 颜色 | `_renderRoad` 内 `rgba(255,255,255,0.30)` / `50 * s` |
| 绿中线宽度 / 颜色 | `_renderRoad` 内 `#7ECB50` / `5.5 * s` |
| 开始钮 / 风景留白 | 开始钮 180×86、底边距屏底 `cfg.startButton.bottom`；改需同步 `bottomReserve` 公式 |

**约束口诀**：`roadSmoothIters` 越大越圆，但偏离关卡中心越多（iter=2 偏移 17px 已接近路面边界 25px，加 3 需确认钮仍压路）；钮坐标改动后必须重跑 `resample_uniform.js` 并重新编译。

## 九、验证记录

- 所有改动均过 `node --check` 语法校验。
- **`tools/sim_verify_fix.js`**（间距三修）：跨 N=12/39/47/53/100 模拟，L26-27、L39-40 恢复正常 ~104px，顶留白精确 100px。
- **`tools/sim_path_verify.js`**（无缝）：Catmull-Rom 穿越所有关卡中心、零 NaN、路径点 275+、每关最大偏移 0.00px。
- **`tools/sim_smooth.js`**（圆角）：Chaikin 各档（0/1/2/3）对照 —— 最尖转角 29.5°→18.8°→13.9°，关卡最大偏移 0→15→17→19px（均 < 路半宽 25px）。
- **点击状态机桩测试**：
  - 点 cleared → 选中按压态 + 触发缩放动画 → 松手回调 `onSelectLevel(0,'cleared')`。
  - 点 locked → 不进按压态（忽略）。
  - 小幅移动（<12px）→ 不滚地图、保持按压。
  - 大移（>12px）→ 取消按压、转滚动。
  - 切 `gameState='playing'` → 选中/进关态自动清理，呼吸环随菜单淡出不残留。
