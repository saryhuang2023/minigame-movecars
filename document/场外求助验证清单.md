## 1. 正常关回归（最关键）
- [ ] 正常进一关 → 通关 → VictoryPopup 正常出现、提示上传正常、关卡地图标记 cleared
- [ ] 从设置面板「home」回菜单后**再打一关**，结算依旧正常（验证 `_resetHelpState` 生效，`playMode` 未被残留 `'assist'` 污染，否则会静默跳过提示上传 + `_markCleared`）
- [ ] 底栏 +3 / 提示 / 求助 三钮在正常关正常渲染且可点

## 2. 发起求助（normal → createHelpRequest）
- [ ] 正常关内点「求助」→ 弹授权（`wx.createUserInfoButton` 取昵称/头像）→ 调 `createHelpRequest` → 拿到 helpKey
- [ ] 分享卡片发出（query 带 `hk`）；当前 `imageUrl` 为占位（已注释，可接受，决策④固定运营图待补）

## 3. assist 流程（好友协助）
- [ ] 冷启：杀进程后用分享卡片打开 → `getLaunchOptionsSync().query.hk` 命中 → 直接进 assist 关
- [ ] 热启：小程序在前台时从卡片/会话打开 → `wx.onShow` query.hk 命中 → 进 assist 关（若当时在别处，应先回菜单再进）
- [ ] assist 关内：底栏 +3 / 提示 / 求助 **不显示、不可点**（防重触发 `_startExternalHelp`、防消耗真实玩家道具）
- [ ] assist 自动录制：走一步录一步；通关后**不弹 VictoryPopup**、结算不污染本人数据
- [ ] 通关后覆盖层显「你帮好友过了这关！」+ 橙钮「发送给好友」→ 分享 hk
- [ ] 提交：`_submitAssist` → 服务端 zlib 压缩 + push assistEntry → 返回 `{code:0}`

## 4. replay 流程（回放协助录制）
- [ ] 主菜单点「求助记录」→ 列表显该关 + 每位协助者「回放N」钮
- [ ] 点「回放」→ `_enterReplayFromHelpKey(hk, idx)` → 进入 replay 关
- [ ] replay 播放中：底栏不可点；常驻灰「返回」钮；**不允许快进**（决策⑥）
- [ ] 播放完显「再看一次」橙钮 + 常驻「返回」；「再看一次」重播原始录制
- [ ] 「返回」→ 回菜单（`_returningToMenu` 防闪引导手）

## 5. 列表 / 续转
- [ ] 列表行状态正确：求助中(n/3) / full(满3人) / expired(过期)
- [ ] 「续转」蓝钮 → `shareAppMessage({query:'hk='+hk})`（同关可多次、每点发一好友、≤3 人）
- [ ] 列表回放 idx 与 `assists` 数组映射正确（多人时回放的是对应那位）

## 6. 服务端边界校验（看云函数返回码）
- [ ] 第 4 人协助 → `code 3`（已满 3 人）
- [ ] 过期求助 → status `expired`，进 assist 被拦截提示已过期
- [ ] 同一 openid 重复协助 → `code 5`
- [ ] helpKey 不存在 → `code 2`

## 7. 压缩契约（已复核）
- 上行：客户端 `JSON.stringify`（**非** deflate）；服务端 `node zlib.deflateSync` 压缩
- 下行：客户端仅 `compress.inflateJson` 解压
- 存储格式 `BASE64(DEFLATE(...))` 不变
